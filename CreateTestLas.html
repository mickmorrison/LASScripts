This script creates a synthetic LiDAR dataset with an assigned Coordinate Reference System (CRS) for development and testing purposes.()

library(lidR)
library(sf)
Linking to GEOS 3.13.1, GDAL 3.11.0, PROJ 9.6.1; sf_use_s2() is TRUE

Attaching package: 'sf'
The following object is masked from 'package:lidR':

    st_concave_hull
library(rgl)

## 1  generate synthetic cloud
set.seed(1)
n_pts <- 1000
df <- data.frame(
  X = runif(n_pts, 0, 100),
  Y = runif(n_pts, 0, 100),
  Z = runif(n_pts, 0, 30),
  Intensity       = sample(1:255, n_pts, TRUE),
  ReturnNumber    = 1L,
  NumberOfReturns = 1L,
  Classification  = 1L
)

## 2  build LAS & patch header
las <- LAS(df)
Creation of a LAS object from data but without a header:
Scale factors were set to 0.001 and XYZ coordinates were quantized to fit the scale factors.
las@header@PHB$`File Source ID` <- 1L
las@header@PHB$`Version Major`  <- 1L
las@header@PHB$`Version Minor`  <- 2L
projection(las) <- 28356               # GDA94 / MGA 56

## 3  save & display (widget auto-prints)
writeLAS(las, "synth_las.las", index = TRUE)
plot3d(las$X, las$Y, las$Z, col = las$Intensity, size = 3)
rglwidget()
